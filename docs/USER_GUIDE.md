## 快速入门

### 首次登录

1. 访问 http://localhost:3000
2. 使用管理员账号登录：
   - 用户名：`admin`
   - 密码：查看后端启动日志或 `.env` 中的 `ADMIN_INITIAL_PASSWORD`
3. 登录后建议立即修改密码（设置 → 账户）

### 配置 AI 模型

1. 进入 **设置 → API 路由**
2. 点击供应商模板（如「小米 MIMO」）或「自定义添加」
3. 填写 API Key 和 API 地址
4. 选择协议模式（推荐「自动探测」）
5. 点击「测试连接」验证
6. 保存

配置完成后，AI 对话、文档编辑、工作流等功能即可使用。

## 文档管理

### 创建文档

1. 点击左侧边栏的「新建文档」按钮
2. 输入文档标题
3. 使用富文本编辑器编写内容
4. 内容自动保存（1.5 秒防抖）

### 编辑器功能

| 功能 | 操作 |
|------|------|
| 标题 | 输入 `#` + 空格，或使用工具栏 |
| 列表 | 输入 `- ` 或 `1. ` |
| 代码块 | 输入 ` ``` ` |
| 引用 | 输入 `> ` |
| 粗体 | `Ctrl+B` |
| 斜体 | `Ctrl+I` |
| 链接 | `Ctrl+K` |
| 图片 | 拖拽或粘贴 |
| 任务列表 | 工具栏按钮 |

### 版本历史

1. 打开文档
2. 点击右上角的「版本」按钮
3. 查看历史版本列表
4. 点击「恢复」回滚到指定版本

### 文件夹管理

1. 左侧边栏显示文件夹树
2. 右键文件夹可重命名、删除、创建子文件夹
3. 拖拽文档到文件夹可移动

### 标签管理

1. 在文档详情页添加标签
2. 左侧边栏显示所有标签
3. 点击标签筛选文档

### 批量导入

1. 进入 **导入** 页面
2. 拖拽或选择文件（支持 Markdown、DOCX、PDF、LaTeX、TXT）
3. 系统自动解析文件结构
4. 点击「AI 分析」获取建议标签和分类
5. 编辑确认后点击「批准导入」

**支持的格式特性**:

| 格式 | 标题 | 粗体/斜体 | 列表 | 表格 | 图片 |
|------|------|-----------|------|------|------|
| Markdown | ✅ | ✅ | ✅ | ✅ | ✅ |
| DOCX | ✅ | ✅ | ✅ | ✅ | ✅ |
| PDF | ✅ (字体检测) | ❌ | ❌ | ❌ | ❌ |
| LaTeX | ✅ | ✅ | ✅ | ❌ | ❌ |

### 导出文档

1. 打开文档
2. 点击「导出」按钮
3. 选择格式：Markdown / HTML / LaTeX / DOCX

## AI 功能

### 智能对话

1. 进入 **AI 助手** 页面
2. 输入问题
3. AI 会自动搜索知识库相关内容作为上下文
4. 支持连续对话

**RAG 检索增强**:
- AI 会自动从知识库中检索相关文档片段
- 检索结果显示在对话下方
- 关闭后可点击「查看检索来源」重新打开

### AI 文本编辑

1. 在编辑器中选中文本（或不选则编辑全文）
2. 点击工具栏的 AI 按钮
3. 选择操作：
   - **润色** — 改善清晰度和流畅性
   - **扩展** — 补充细节和示例
   - **压缩** — 精简到 50%
   - **翻译** — 中英互译
   - **修正** — 修复语法错误

### AI 文档生成

1. 进入文档列表
2. 点击「AI 生成」按钮
3. 输入需求描述
4. 选择文档类型（教程/报告/方案/笔记/参考）
5. AI 自动生成结构化文档

### 文档分析

1. 在导入页面或文档详情页
2. 点击「AI 分析」
3. 获取：
   - 结构化摘要
   - 关键词提取
   - 标签推荐
   - 质量评分
   - 改进建议

1. 进入 **工作流** 页面

   - 从左侧面板拖拽节点到画布
   - 连接节点的输入/输出端口
   - 点击节点配置参数

### 节点类型

|------|------|------|

| 处理 | 内容扩展 | 补充细节 |
| 处理 | 精简压缩 | 压缩内容 |
| 处理 | 翻译 | 中英互译 |
| 处理 | 修正语法 | 修复错误 |
| 处理 | 生成摘要 | AI 摘要 |
| 处理 | 提取关键词 | 关键词提取 |
| 处理 | 自动打标签 | 自动标签 |
| 处理 | 标准化分析 | 质量分析 |
| 处理 | AI 深度分析 | 综合分析 |
| 处理 | 格式转换 | 格式转换 |
| 处理 | 条件分支 | 条件判断 |
| 处理 | 循环处理 | 批量处理 |
| 处理 | 人工审批 | 暂停等待审批 |

| 输出 | 导出文档 | 导出文件 |

### 执行工作流

1. 配置完节点后点击「执行」
2. 选择输入文档范围
3. 监控执行进度
4. 查看每个节点的执行结果

### 定时执行

1. 打开工作流设置
2. 配置 Cron 表达式（如 `0 9 * * 1` = 每周一 9 点）
3. 启用定时任务

## API 路由配置

### 添加端点

1. 进入 **设置 → API 路由**
2. 选择供应商模板或自定义添加
3. 填写配置：
   - **端点名称**: 自定义标识
   - **API Key**: 提供商的 API 密钥
   - **API 地址**: 默认自动填充
   - **支持的模型**: 选择或自定义模型名
   - **优先级**: 数字越小优先级越高
   - **协议模式**: 自动探测 / Chat Completions / Responses

### 协议模式说明

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 自动探测 | 首次测试时自动检测 | 推荐默认选项 |
| Chat Completions | `/v1/chat/completions` | 大多数兼容提供商 |
| Responses | `/v1/responses` | 仅 OpenAI 原生 |

### 多端点智能调度

系统支持配置多个端点，自动进行智能调度：

- **健康监控**: 实时监控端点状态
- **故障转移**: 端点失败自动切换到备用端点
- **优先级排序**: 按优先级和健康评分排序
- **自动冻结**: 连续失败 5 次自动冻结，渐进式恢复
- **连接池**: HTTP 客户端缓存复用

### 健康面板

在 API 路由页面查看：
- 端点总数、健康/降级/冻结/禁用数量
- 每个端点的成功率、延迟、请求次数
- 最近错误信息
- 冻结状态和恢复时间

## MCP 集成

### 什么是 MCP？

MCP（Model Context Protocol）是一个开放协议，让 AI 工具（如 Claude Desktop、Cursor、Windsurf）可以直接与知识库交互。

### 配置 Claude Desktop

1. 打开 Claude Desktop 设置
2. 编辑 `~/.claude/claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "knowledge-base": {
      "command": "python",
      "args": ["-m", "app.mcp.stdio_server"],
      "cwd": "/path/to/knowledge-base/backend"
    }
  }
}
```

3. 重启 Claude Desktop
4. 在对话中即可使用知识库工具

### 配置 Cursor

1. 打开项目根目录的 `.cursor/mcp.json`
2. 添加同样的配置

### 使用示例

在 Claude Desktop 中：

```
用户: 帮我搜索知识库中关于 Python 的文档
Claude: [调用 search_documents 工具] 找到了 5 篇相关文档...

用户: 把搜索结果整理成一份摘要文档
Claude: [调用 read_document 读取文档] [调用 create_document 创建摘要]

用户: 分析一下文档 #42 的质量
Claude: [调用 analyze_document] 文档质量评分 85/100，建议...
```

### MCP 工具列表

**只读工具**:
- `search_documents` — 语义/关键词搜索
- `read_document` — 读取文档内容
- `list_documents` — 列出文档
- `get_document_versions` — 版本历史
- `get_folder_tree` — 文件夹树
- `summarize_document` — AI 摘要
- `analyze_document` — AI 分析
- `search_knowledge_graph` — 知识图谱搜索
- `list_folders` — 文件夹列表
- `list_tags` — 标签列表

**写入工具**:
- `create_document` — 创建文档（默认 draft 状态）
- `update_document` — 更新文档（自动版本快照）
- `delete_document` — 删除/归档文档
- `move_document` — 移动文件夹/改状态
- `manage_tags` — 添加/移除标签

### MCP 资源

AI 工具可以发现和读取这些资源：
- `kb://documents` — 文档列表
- `kb://documents/{id}` — 单个文档
- `kb://folders` — 文件夹
- `kb://tags` — 标签
- `kb://stats` — 统计

### MCP 提示模板

预定义的交互模板：
- `review_document` — 文档审查
- `summarize_topic` — 主题总结
- `compare_documents` — 文档对比
- `write_from_requirements` — 按需求写作
- `organize_knowledge_base` — 知识库整理

### 安全说明

- **Stdio 传输**: 本地进程，无需额外认证
- **SSE 传输**: 需要 JWT Bearer Token 认证
- **写入操作**: 默认创建 draft 状态，需人工审核发布
- **无删除权限**: MCP 无法永久删除文档（仅归档）

## 常见问题

### Q: AI 对话没有响应？

A: 检查：
1. 是否在 API 路由中配置了端点
2. 端点测试是否通过
3. API Key 是否有效
4. 后端是否正常运行

### Q: 导入 DOCX 后格式丢失？

A: 确保使用最新版本，DOCX 导入已支持标题、粗体、列表、表格等格式保留。

### Q: MCP 工具无法连接？

A: 检查：
1. 后端是否运行在 8000 端口
2. 配置文件中的路径是否正确
3. Python 环境是否安装了 `mcp` 包

### Q: 如何切换 AI 模型？

A: 进入 **设置 → API 路由**，添加或编辑端点，选择不同的模型。

### Q: 文档丢失了怎么办？

A: 检查：
1. 是否被归档（状态为 archived）
2. 是否在回收站中
3. 通过版本历史恢复
